PHK Bank

PHK Bank는 예치·결제·송금·보상·기본소득(UBI)을 하나의 흐름으로 묶어 PWB( Pi World Bank ) 생태계에서 실행되는 금융 OS 역할을 합니다.
사용자는 “맡기고(예치) → 쓰고(결제) → 보내고(송금) → 받고(보상/UBI)”를 한 화면에서 처리합니다.

PWB Ecosystem Deposit • Pay • Transfer Rewards • UBI Staking / Vault

상태

모바일 클릭/설명 방식 적용 (hover 제거)

지금 페이지는 “눌러도 반응 없음”을 막기 위해
유튜브/틱톡을 onclick이 아닌 링크(a)로 고정했습니다.

빠른 이동

기능 유튜브/틱톡 스테이킹

핵심 4기능 (클릭하면 설명이 열립니다)

결제

Pi 기반 즉시 결제, 정산 흐름을 단순화합니다.

결제는 PHK Bank의 핵심 사용 시나리오입니다.
1) 사용자는 Pi로 결제합니다. 2) 가맹점은 즉시 확인합니다. 3) 정산은 자동으로 기록됩니다.
포인트: “결제 버튼을 누르면 설명이 바로 뜨도록” 모바일 클릭 구조로 만들었습니다.

송금

지갑 간 송금 흐름을 한 번에 안내합니다.

송금은 “주소 입력 → 금액 입력 → 확인”의 3단계로 단순화합니다.
거래는 기록되며, 사용자에게는 “최근 거래” 형태로 요약 표시됩니다.
주의: 실제 온체인 송금은 SDK/서버 연동이 필요합니다. 본 페이지는 UI/설명 구조입니다.

예치

예치율·보상·정산 구조를 설명합니다.

예치는 “자산 안정 + 생태계 유동성”을 동시에 만드는 장치입니다.
예치된 물량은 PWB 생태계에서 결제/정산/보상 흐름의 기반이 됩니다.
예치율은 정책에 따라 조정 가능하며, 보상은 “기여 기반”으로 설계할 수 있습니다.

보상

참여·사용·기여에 따른 보상 흐름을 만듭니다.

보상은 “사용/기여/정산 참여”에 대한 인센티브입니다.
예: 결제 참여, 가맹점 유지, 설문/검증, 커뮤니티 기여 등에 따른 보상.
목표: 단기 펌핑이 아니라 “지속 가능한 유입/유지” 구조로 설계합니다.

기본소득(UBI)

PWB 수익·정책 기반 UBI 분배 구상입니다.

UBI는 “최종 수혜자(서민/노동자/노년층/청년/소상공인)”에게 돌아가는 구조입니다.
지급 방식은 월/주/일 단위로 설계 가능하며, “기여 기반 + 최소 보장” 혼합 모델도 가능합니다.

스테이킹

스테이킹 참여자 보호·재스테이킹 옵션을 제공합니다.

문제: “3만4천개 스테이킹” 참여자들이 기대와 달리 멈춰버리면 참담하고 실망할 수 있습니다.
해법: PWB 안에서 추가 스테이킹(재스테이킹) 또는 기여 기반 보상 전환 같은 “출구/유지” 설계를 제공합니다.
핵심 메시지: “스테이킹은 끝이 아니라, PWB 내에서 다시 굴리는 구조”입니다.

투명성

정책·보상·예치율을 공개형으로 운영합니다.

정책 변경(예치율/보상/UBI)은 공지와 버전으로 관리합니다.
사용자는 “왜 이렇게 되는지”를 화면에서 바로 확인할 수 있어야 합니다.

유튜브 · 틱톡 (눌러도 반응 없던 문제 해결)

모바일/Pi Browser에서 onclick + window.open은 자주 막힙니다.
그래서 아래는 JS 없이 a 링크로 고정했습니다. (반응 100% 나옵니다)

YouTube 열기 TikTok 열기
중요: 위 주소의 YOUR_CHANNEL 과 YOUR_TIKTOK_ID만 대표님 계정으로 바꿔주시면 “누르면 바로 대표님 영상”으로 이동합니다.
내 유튜브 영상 채널/플레이리스트 ID만 교체
내 틱톡 영상 임베드가 막히면 위 링크 사용
틱톡은 보안 정책 때문에 앱/브라우저 환경에서 임베드가 막히는 경우가 많습니다.
그래서 상단의 TikTok 열기 링크를 “확실한 방식”으로 넣어두었습니다.

PHK Bank ↔ PWB 관계 (요약)

PHK Bank는 “사용자 화면(금융 OS)”이고, PWB는 “생태계(예치·결제·보상·UBI가 돌아가는 전체 시스템)”입니다.
즉, PHK Bank는 PWB의 실행 앱/포털이며, PWB는 PHK Bank가 작동하는 경제권입니다.

한 줄로: PHK Bank는 “앞단 UI”, PWB는 “뒤단 경제 시스템”입니다.
사용자가 느끼는 모든 체감(결제, 송금, 보상, UBI)은 PHK Bank에서 시작하지만, 실제 구조(예치율, 보상정책, UBI 분배)는 PWB의 정책/엔진에서 돌아갑니다.

대표님이 “반응 없음”을 다시 안 겪게 하려면

1) 버튼은 되도록 a 링크로 연결합니다.
2) 설명은 hover가 아니라 클릭 토글로 만듭니다.
3) Pi Browser/모바일은 window.open을 피합니다.
